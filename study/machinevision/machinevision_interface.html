인터페이스 주변기기 (Interface) - 심화

머신비전 개발자를 위한 인터페이스 · 드라이버 심화 학습

1. 개요

  • 인터페이스 주변기기는 카메라와 컴퓨팅 플랫폼 사이에서 이미지 데이터를 전달하는 역할을 담당
  • 사용하는 케이블 규격(Ethernet, USB, CoaXPress, Camera Link)에 따라 필요한 주변기기가 달라짐 (케이블/커넥터 자체의 물리 스펙은 케이블 페이지 참고)
  • 인터페이스 주변기기의 선택은 전송 속도, 케이블 길이, 다중 카메라 구성 여부 등을 고려하여 결정
개발자 노트 — 이 페이지는 "PC/컴퓨팅 플랫폼 쪽에서 카메라 데이터를 받기 위해 무엇을 꽂아야 하는가"에 초점을 둡니다. 케이블 자체의 대역폭·길이·전원 스펙은 케이블 연결 페이지를 함께 참고하세요.

2. Ethernet 인터페이스

① 네트워크 인터페이스 카드 (NIC, Network Interface Card)

  • PCIe 슬롯을 통해 컴퓨터 마더보드에 연결하는 확장 카드
  • OSI 모델의 물리 계층 및 데이터 링크 계층에 대한 전자 회로를 구현
  • 고속 GigE 카메라 연결에 필수적이며, 마더보드 내장 이더넷 포트 대비 높은 버퍼 크기와 성능을 제공
개발자 노트 — GigE Vision 카메라 성능이 이론치보다 낮게 나온다면 NIC 설정을 먼저 점검하세요.
· 점보 프레임(Jumbo Frame) : 표준 이더넷 MTU(1500바이트) 대신 최대 9000바이트로 설정하면 큰 이미지 전송 시 패킷 수와 오버헤드가 줄어 처리율이 향상됨
· 수신 버퍼(Receive Descriptor) 크기 : NIC 드라이버 설정에서 수신 버퍼 개수를 최대치로 늘리면 프레임 드랍을 줄일 수 있음 (단, 시스템 메모리 사용량 증가)
· 여러 대의 GigE 카메라를 동시에 쓸 경우, 카메라마다 별도의 NIC 포트를 할당하는 것이 대역폭 경합을 피하는 가장 확실한 방법

② 스위치 (Switch)

  • 동일한 LAN(Local Area Network) 내에서 이더넷 장치를 연결하는 장비
  • 수신한 패킷의 헤더를 읽어 대상 주소를 파악하고 해당 포트로만 전달 (브로드캐스트 없음)
  • 머신 비전에서 스위치가 필요한 경우
    • 멀티플렉싱 : 고속 멀티 카메라 시스템에서 여러 이더넷 포트의 픽셀 데이터를 통합하여 단일 포트로 출력
    멀티캐스팅 : 카메라 데이터를 여러 호스트 PC로 전송
    PTP(Precision Time Protocol, IEEE 1588) : 여러 카메라의 타임스탬프를 단일 장치 시계와 마이크로초 단위로 동기화 (GigE Vision 2.0 표준 포함)
개발자 노트 — 여러 카메라를 "동시에 찍은 것처럼" 다뤄야 하는 스테레오/멀티뷰 검사 시스템에서는 PTP를 지원하는 관리형(Managed) 스위치를 선택하고, 카메라·스위치 모두에서 PTP를 활성화해야 타임스탬프 기반 동기화가 실제로 동작합니다.

③ 이더넷 허브 (Ethernet Hub)

  • 여러 장치를 연결할 수 있으나, 스위치와 달리 연결된 모든 장치로 패킷을 브로드캐스트
  • 스위치 대비 효율이 낮아 현재 머신 비전 환경에서는 스위치를 주로 사용

④ 마더보드 내장 이더넷 포트

  • 대부분의 마더보드에 기본 탑재되어 있으며, 주로 1Gbps를 지원 (일부 5Gbps / 10Gbps)
  • 제한된 내부 버퍼 크기와 낮은 성능으로 인해 고속 머신 비전 애플리케이션에는 권장하지 않음

3. USB 인터페이스

① USB PCIe 카드

  • PC의 USB 포트 수를 늘리는 PCIe 기반 확장 카드
  • USB 2.0, USB 3.0 등 다양한 표준과 USB-A, USB-B, USB-C, Micro-USB 커넥터를 지원하는 제품이 시판 중

② USB 포트 허브

  • 컴퓨터에 연결하여 USB 포트를 추가 확장하는 외부 장치
  • 추가 USB 장치 연결이 필요할 때 사용

③ 마더보드 내장 USB 포트

  • 요즘 대부분의 마더보드에는 USB 3.0 포트가 내장되어 있으며, 일부는 USB 2.0과 USB 3.0을 함께 지원
개발자 노트 — USB는 컨트롤러 칩셋에 따라 대역폭을 여러 포트가 나눠 쓰는 경우가 많습니다. 멀티 카메라 USB3 시스템을 구성할 때는 포트별로 독립된 컨트롤러(별도 PCIe 카드의 개별 컨트롤러)를 쓰는지 확인해야 실제 대역폭 경합을 피할 수 있습니다.

4. CoaXPress 인터페이스

① 프레임 그래버 (Frame Grabber)

  • 카메라에서 개별 프레임을 캡처하여 처리하는 전용 확장 카드
  • 주요 기능
    • 실시간 압축
    관심 영역(ROI) 오프로드로 추가 대역폭 활용
    다양한 고급 획득 및 이미지 처리 기능 지원
개발자 노트 — 프레임 그래버는 PCIe 슬롯의 레인 수(x4, x8 등)만큼 대역폭이 제한됩니다. 여러 장의 프레임 그래버를 한 시스템에 꽂을 경우, 메인보드의 PCIe 슬롯이 CPU에 직결된 레인인지, 칩셋을 경유해 대역폭을 나눠 쓰는 슬롯인지 반드시 확인해야 합니다.

5. Camera Link / CLHS 인터페이스

① 프레임 그래버 (Frame Grabber)

  • Camera Link 카메라에서 프레임을 취득하는 전용 확장 카드
  • 주요 온보드 처리 기능
    • 프로그래밍 가능한 관심 영역(ROI)
    픽셀 데시메이션 (Pixel Decimation)
    이미지 스케일링 (Image Scaling)
    조회 테이블 (Lookup Table)

6. 표준 드라이버 계층 : GenICam / GenTL

  • GenICam(Generic Interface for Cameras) : GigE Vision, USB3 Vision, CoaXPress, Camera Link 등 서로 다른 인터페이스 기술을 하나의 공통 API로 다룰 수 있게 해주는 표준 (EMVA 관리)
  • GenTL(Generic Transport Layer) : 카메라 검색, 이미지 스트리밍, 이벤트 처리 등 하드웨어 종속적인 부분을 담당하는 전송 계층. 카메라·프레임그래버 제조사가 .cti 파일(GenTL Producer)로 제공
  • GenApi / SFNC : 노출, 게인 같은 카메라 기능의 이름과 동작 방식을 표준화해, 제조사가 달라도 동일한 방식으로 파라미터에 접근 가능
개발자 노트 — 애플리케이션을 GenICam/GenTL 표준 위에서 개발하면, 특정 제조사의 전용 SDK에 종속되지 않고 카메라 모델을 교체해도(GigE ↔ USB3 ↔ CXP 포함) 애플리케이션 코드를 거의 그대로 재사용할 수 있습니다. 신규 시스템을 설계할 때는 제조사 전용 SDK보다 GenICam/GenTL 호환 여부를 먼저 확인하는 것을 권장합니다.

7. 멀티 카메라 동기화 & 성능 튜닝

튜닝 항목내용
점보 프레임(Jumbo Frame)MTU를 1500 → 최대 9000바이트로 설정, 패킷 수·오버헤드 감소
NIC 수신 버퍼수신 디스크립터 개수를 최대로 늘려 프레임 드랍 방지
PTP(IEEE 1588)여러 카메라의 타임스탬프를 마이크로초 단위로 동기화
CPU/IRQ 친화도고속 스트리밍 카메라의 인터럽트를 특정 CPU 코어에 고정해, 다른 프로세스와의 자원 경합을 줄임
PCIe 대역폭 확인프레임그래버·NIC가 CPU 직결 레인인지, 칩셋 공유 레인인지 확인
개발자 노트 — 단일 카메라에서는 문제없던 시스템이 카메라 대수를 늘리면 프레임 드랍이 생기는 경우, 대부분 원인은 알고리즘이 아니라 위 표의 네트워크/PCIe 튜닝 항목에 있습니다. 카메라를 추가하기 전에 총 필요 대역폭(카메라 페이지의 대역폭 계산식 참고)과 시스템의 실제 여유 대역폭을 먼저 비교해보는 것이 좋습니다.

8. 머신비전 개발자를 위한 인터페이스 선정 체크리스트

  • 총 필요 대역폭 — 전체 카메라 수 × 카메라별 필요 대역폭이 NIC/프레임그래버 총 처리량 이내인지 확인
  • 동기화 요구사항 — 여러 카메라를 시간적으로 맞춰야 한다면 PTP 지원 스위치·카메라 선택
  • 표준 준수 — GenICam/GenTL 호환 여부로 향후 카메라 교체·유지보수 유연성 확보
  • PCIe 슬롯 구성 — 프레임그래버·다중 NIC를 꽂을 메인보드의 PCIe 레인 배분 확인
  • 드라이버/OS 지원 — Windows/Linux 등 실제 운영 환경에서 제조사 드라이버·SDK 지원 여부
  • 확장성 — 향후 카메라 대수 증가를 감안한 여유 포트·슬롯 확보
