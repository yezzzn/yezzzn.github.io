카메라 (Camera) - 심화

머신비전 개발자를 위한 이미지 센서 · 파라미터 심화 학습

1. 카메라 시스템 구성

  • 렌즈 마운트, 이미지 센서, 영상처리 프로세서(ISP/FPGA), 전력 전자 장치, 통신 인터페이스 커넥터를 포함하는 보호 케이스로 구성
  • 핵심은 이미지 센서 : 들어오는 빛(광자)을 전기 신호(전자)로 변환(광전 효과)하는 소자
  • 이미지 센서를 구성하는 핵심 요소
    "포토다이오드(Photodiode)" : 반도체 소자로, 입사된 광자를 전자로 변환해 전하 형태로 저장하는 "포텐셜 우물(Potential Well)" 역할을 함
    "ADC(Analog-to-Digital Converter)" : 포토다이오드가 축적한 아날로그 전압 값을 디지털 값(그레이 레벨)으로 변환
  • 이미지 센서 분류 기준
    물리적 구조 : CCD, CMOS
    픽셀 배열 : Area Scan(면 스캔), Line Scan(선 스캔)
    채도 유형 : Color, Mono
    셔터 유형 : Global Shutter, Rolling Shutter
    감응 스펙트럼 : UV, Visible, NIR, SWIR
  • 포토다이오드 1개 = 디지털 이미지의 픽셀 1개에 대응
  • 포토다이오드는 아날로그 전하 값을 가지며, ADC를 거친 후에야 픽셀은 디지털 값을 가짐
  • 픽셀(Pixel)은 디지털 이미지를 구성하는 최소 단위이며, "Picture Element"의 약자
    • 머신비전 카메라의 일반적인 하드웨어 구성
개발자 노트 — 카메라는 검사 알고리즘 파이프라인의 첫 단계, 즉 "광학 신호 → 디지털 데이터" 변환기입니다. 센서 특성(노이즈, 다이내믹 레인지, 셔터 방식)을 이해해야 임계값 설정, 노이즈 필터링, ROI 설계 같은 후단 알고리즘의 근거를 세울 수 있습니다.

2. 이미지 센서 아키텍처 : CCD vs CMOS

  • CCD(Charge-Coupled Device) : 모든 픽셀의 전하를 순차적으로 하나의 공통 출력 앰프까지 이동시켜 직렬로 판독
  • CMOS(Complementary Metal-Oxide-Semiconductor) : 각 픽셀(또는 컬럼) 단위로 증폭기와 ADC를 내장해 병렬로 판독
구분CCDCMOS
판독 방식단일 출력단으로 직렬 판독픽셀/컬럼 단위 병렬 판독
속도 / 전력상대적으로 느림, 전력 소모 큼고속, 저전력
노이즈 균일성우수 (Fixed Pattern Noise 적음)과거 대비 크게 개선, 최근은 CCD 수준
비용상대적으로 고가대량 생산 용이, 저렴
현재 산업 표준일부 고감도 과학용 카메라대부분의 산업용 머신비전 카메라
개발자 노트 — 현재 신규 라인 설계 시 대부분 CMOS Global Shutter 센서가 표준입니다. CCD는 극저조도·고정밀 정량 계측(과학 이미징) 용도로 남아 있는 경우가 많습니다. 센서 스펙 문서에서 "Sensor Type"과 "Shutter Type"을 먼저 확인하는 습관이 필요합니다.

3. 픽셀 배열 구조 : Area Scan vs Line Scan

  • Area Scan (면 스캔) : 픽셀이 가로×세로 2차원 매트릭스로 배열된 센서. 한 번의 노출로 전체 2차원 이미지를 한 프레임에 그대로 획득함
    • 정지된 개별 부품 검사, PCB 검사, 로봇 픽앤플레이스 등 "한 장면을 통째로" 찍는 대부분의 머신비전 애플리케이션에 사용
  • Line Scan (선 스캔) : 픽셀이 한 줄(1차원)로만 배열된 센서. 한 번에 한 줄(라인)만 캡처하며, 카메라와 대상물 사이에 지속적인 상대 이동이 있어야 여러 줄을 이어 붙여 하나의 2차원 이미지를 완성할 수 있음
    • 컨베이어에 연결된 인코더(Encoder) 펄스에 맞춰 라인 캡처 타이밍을 동기화하는 것이 일반적
    압연 강판, 필름/시트, 인쇄물 웹(Web) 검사, 회전하는 롤·파이프 표면 전개 검사처럼 "연속적으로 흘러가는" 대상에 사용
구분Area ScanLine Scan
센서 배열2차원 매트릭스1차원(단일 줄)
이미지 획득한 번의 노출로 프레임 전체 획득줄 단위로 반복 캡처 후 이어붙임
필요 조건없음 (정지 상태로 촬영 가능)카메라-대상물 간 지속적 상대 이동, 인코더 동기화
대표 용도개별 부품 검사, PCB 검사, 로봇 비전웹/필름/강판 검사, 롤·파이프 표면 전개 검사
셔터 개념과의 관계Global/Rolling Shutter 선택 가능한 줄 내 픽셀은 동시 노출(개념상 Global)이 기본이며, 별도로 "Rolling Shutter"라는 용어는 통상 사용하지 않음
개발자 노트 — 두 개념이 헷갈릴 때는 "몇 차원으로 찍는가(Area/Line) → 그 프레임 안에서 언제 노출되는가(Global/Rolling)" 순서로 질문을 나눠서 생각하면 명확해집니다. Area Scan 카메라를 고를 때만 Global/Rolling Shutter를 추가로 고려하면 됩니다.

4. 셔터 방식 : Global Shutter vs Rolling Shutter

※ 앞서 3절의 Area/Line Scan은 "센서 배열 구조"에 대한 분류이고, 아래 내용은 (주로 Area Scan 센서에서) "노출·판독 타이밍"에 대한 분류입니다.
  • Global Shutter : 센서의 모든 픽셀이 동일한 순간에 동시에 노출·판독됨 → 움직이는 물체도 왜곡 없이 촬영
  • Rolling Shutter : 행(Row) 단위로 순차적으로 노출·판독됨 → 이동 중인 물체는 상단과 하단의 노출 시점이 달라 기울어짐(Skew) 왜곡 발생 가능
개발자 노트 (선택 가이드)
· 컨베이어 벨트, 고속 이송, 회전체 검사 → Global Shutter 필수 (측정 왜곡 방지)
· 정지 상태에서 촬영하는 로봇 픽앤플레이스, PCB 검사, 정적 표면 검사 → Rolling Shutter로 비용 절감 가능
· 참고 : Teledyne Vision Solutions, VA Imaging 등 주요 머신비전 벤더의 기술 자료 기준

5. 디지털 이미지 핵심 파라미터

Resolution

① 해상도 (Resolution)

  • 디지털 이미지는 픽셀로 구성되며, 이미지 해상도는 가로·세로 픽셀 수로 표현
    • 1280 × 1024 해상도는 너비 1280픽셀, 높이 1024픽셀을 의미
  • 같은 센서 크기에서 픽셀 수를 늘리면(고해상도) 픽셀 하나의 물리적 크기(Pixel Pitch)는 작아짐
    • 픽셀이 작아지면 픽셀당 받아들이는 광자 수가 줄어 감도(SNR)가 낮아지는 트레이드오프 발생
개발자 노트 — 검출하려는 최소 결함(피처) 크기 대비 최소 2~3픽셀 이상이 걸치도록 해상도/시야(FoV)를 설계하는 것이 실무 기준(나이퀴스트 샘플링 관점). 예를 들어 0.1mm 결함을 검출하려면 픽셀당 실제 크기(GSD)가 0.03~0.05mm 이하가 되도록 카메라·렌즈·작동거리를 산정해야 함.
Bit Depth

② (빛의) 세기와 비트 심도 (Intensity & Bit Depth)

  • 이미지 픽셀의 밝기 값을 세기(Intensity)라고 함
  • ADC의 비트 심도가 클수록 세기를 표현할 수 있는 계조(Gray Level) 범위가 커짐
    • 8비트 → 256단계(0~255), 10비트 → 1024단계, 12비트 → 4096단계
  • 다이내믹 레인지(Dynamic Range) = 센서가 담을 수 있는 최대 신호(Full Well Capacity) 대비 최소 신호(암노이즈)의 비율
  • 비트 심도가 높을수록 밝은 영역과 어두운 영역의 미세한 명암 차이를 함께 표현하기 유리함 (단, 실제 다이내믹 레인지는 센서 자체 성능에 의해 제한됨)
개발자 노트 — 금속 표면처럼 반사가 강해 명암 대비가 큰 대상은 8비트로는 계조 손실(Clipping)이 발생하기 쉬움 → 12비트 이상 센서 + HDR 모드 고려. 반대로 단순 이진화(Threshold) 기반 결함 검출에는 8비트로도 충분한 경우가 많아 불필요한 데이터 대역폭 낭비를 피할 수 있음.
Exposure

③ 노출 (Exposure)

  • 포토다이오드가 빛에 노출되는 시간(Exposure Time / Shutter Speed)
  • 일반적으로 마이크로초(µs) 또는 밀리초(ms) 단위로 설정
  • 노출 시간이 길수록 밝아지지만, 이동 중인 물체는 모션 블러(Motion Blur)가 커짐
    • 모션 블러(px) ≈ 물체 이동 속도(mm/s) × 노출 시간(s) ÷ 픽셀당 실제 크기(mm/px)
개발자 노트 — 고속 라인에서는 노출 시간을 짧게(수십~수백 µs) 가져가는 대신, 부족한 광량은 스트로브(Strobe) 조명 + 트리거 동기화로 보완하는 것이 표준적인 접근입니다. 상시 점등 조명보다 스트로브 방식이 노출 대비 순간 광량 확보와 조명 수명 관리에 유리합니다.
Gain

④ 증폭 (Gain)

  • 픽셀 신호 강도 값을 전기적으로 증폭
  • 센서 단에서 포토다이오드 전하 신호를 증폭하면 아날로그 게인(Analog Gain)
  • ADC 변환 후 디지털 값을 증폭하면 디지털 게인(Digital Gain)
  • 게인을 올리면 밝기와 함께 노이즈도 동일한 비율로 증폭되어 SNR이 저하됨
개발자 노트 — 밝기가 부족할 때 우선순위는 ① 조명 강화 → ② 노출 시간 조정 → ③ 게인 조정 순서가 일반적입니다. 게인은 노이즈를 함께 키우므로 알고리즘의 검출 정확도를 낮출 수 있는 마지막 수단으로 접근하는 것이 안전합니다.
Frame Rate

⑤ 프레임 속도 (Frame Rate)

  • 센서가 1초에 출력할 수 있는 이미지 수 (FPS, Frames Per Second)
  • 필요 대역폭(bps) ≈ 가로 해상도 × 세로 해상도 × 비트 심도 × FPS
    • 해상도·비트심도·프레임레이트가 커질수록 인터페이스(GigE, USB3, CoaXPress 등) 대역폭 한계에 먼저 부딪힘
  • ROI(관심영역) 설정 또는 Binning(인접 픽셀 병합)으로 전송 데이터량을 줄여 프레임레이트를 높이는 기법이 실무에서 자주 사용됨
개발자 노트 — 인터페이스별 대역폭 한계와 트리거 방식은 인터페이스(Interface) 페이지에서 함께 확인하는 것을 권장합니다.

6. 컬러 처리 : Bayer 패턴과 Mono vs Color

  • 일반적인 컬러 센서의 각 포토다이오드 앞에는 R/G/B 중 하나의 색만 통과시키는 컬러 필터 어레이(CFA, Color Filter Array)가 배치됨
  • 가장 널리 쓰이는 배열이 Bayer 패턴이며, 사람 눈이 녹색에 민감한 특성을 반영해 R:G:B = 1:2:1 비율(RGGB)로 구성
  • 각 픽셀은 한 가지 색상 정보만 가지므로, 주변 픽셀 값을 보간(Interpolation)해 나머지 색상을 추정하는 디모자이킹(Demosaicing) 과정을 거쳐 최종 컬러 이미지가 생성됨
구분Mono (흑백)Color (컬러)
해상도 효율모든 픽셀이 밝기 정보를 직접 사용 → 실효 해상도 높음디모자이킹 보간으로 실효 해상도 저하
감도(SNR)컬러 필터로 인한 광량 손실 없음 → 저조도에 유리필터 투과율만큼 광량 손실
주요 용도형상·엣지·명암 기반 결함 검출색상 판별(도장, 라벨, 이물 색상 검사)
개발자 노트 — 색상 정보 자체가 검사 기준이 아니라면(예: 스크래치, 치수, 이물 형상 검출) Mono 센서가 해상도와 감도 모두에서 유리합니다. 색상 판정이 반드시 필요한 경우에만 Color 센서를 선택하는 것이 비용·성능 측면에서 합리적입니다.

7. 머신비전 개발자를 위한 카메라 선정 체크리스트

  • 해상도 & 픽셀 크기 — 검출 대상 최소 결함 크기 대비 충분한 GSD(픽셀당 실제 크기) 확보 여부
  • 셔터 방식 — 대상이 움직이는지 여부에 따라 Global / Rolling 결정
  • 프레임 속도 & 인터페이스 대역폭 — 라인 스피드(Takt Time) 대비 필요 FPS와 인터페이스(GigE/USB3/CoaXPress) 대역폭이 충분한지 확인
  • 트리거 / GPIO 지원 — 외부 센서·PLC와의 하드웨어 트리거, 스트로브 출력 지원 여부
  • 렌즈 마운트 호환성 — C-mount, CS-mount, F-mount 등 렌즈와의 물리적 호환성
  • SDK / 표준 프로토콜 지원 — GenICam, GigE Vision, USB3 Vision 등 표준 준수 여부 (개발 생산성에 직결)
  • 설치 환경 내성 — 진동, 분진, 온도 범위 등 실제 설치 환경에 대한 스펙(IP 등급) 충족 여부
