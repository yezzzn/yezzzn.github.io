컴퓨팅 플랫폼 (Computing Platform) - 심화

머신비전 개발자를 위한 컴퓨팅 플랫폼 선정 심화 학습

1. 개요

  • 머신 비전 카메라 내부의 프로세서(임베디드 프로세서 또는 FPGA)는 이미지 센서에서 픽셀 값을 읽고, 이미지를 처리하여 외부 장치로 출력하는 역할을 담당
  • 카메라 내부 프로세서는 전력 소비와 발열을 최소화하는 방향으로 설계되어, 매우 높은 프레임 속도의 후처리에는 처리 능력이 제한적임
  • 고속·고해상도 카메라에서 나오는 대량의 이미지 데이터를 후처리할 경우, 외부 컴퓨팅 플랫폼으로 데이터를 보내 처리하는 것이 권장됨
개발자 노트 — 컴퓨팅 플랫폼 선정은 결국 "카메라에서 초당 쏟아지는 데이터량(카메라 페이지의 대역폭 계산식 참고)을 실시간으로 받아, 요구되는 알고리즘(단순 이진화 ~ 딥러닝 추론)을 목표 사이클 타임 안에 처리할 수 있는가"의 문제입니다. 처리 성능뿐 아니라 설치 환경 내구성, 확장성, 비용까지 함께 고려해야 합니다.

2. 컴퓨팅 플랫폼의 종류

① 소비자 PC (Consumer PC)

  • 가장 유연하고 저렴한 옵션으로, 기성 구성 요소를 사용하여 시스템을 설계 가능
  • 이더넷, USB, CoaXPress, Camera Link 등 다양한 인터페이스로 머신 비전 카메라와 연결 가능

② 산업용 PC (Industrial PC)

  • 먼지, 이물질 차단을 위한 견고한 케이스에 내장
  • 충격, 진동, 극한 온도 등 산업 환경의 혹독한 조건을 견딜 수 있도록 설계
  • 소비자 PC와 동일한 인터페이스(이더넷, USB 등)로 카메라와 연결 가능

③ 비전 컨트롤러 (Vision Controller)

  • PC 기능에 더해 직렬 및 I/O 장치 지원을 추가로 제공하는 전용 시스템
  • 내장 인터페이스
    • RS232 / RS485 포트
    아날로그 I/O 및 디지털 I/O
  • PLC(프로그래밍 가능 로직 컨트롤러) 및 산업용 통신 모듈과 쉬운 통합 가능
  • Modbus 직렬과 같은 통신 프로토콜 구현 지원, 일부 제품은 조명 제어를 위한 PWM 기능 탑재

④ 임베디드 시스템 (Embedded System)

  • 자율 주행 차량, 드론, 의료 기기 등 다양한 소형 장치·시스템에 내장할 수 있도록 설계된 고도로 구성 가능한 초소형 컴퓨터
  • 이더넷 및 USB 등 표준 인터페이스를 지원하여 머신 비전 카메라와 쉽게 연동 가능
  • 대표 제품 : NVIDIA Jetson AGX Orin(최대 275 TOPS, GPU+DLA 내장), Jetson Orin Nano(경량·저전력형)
개발자 노트 — Jetson 계열은 CPU, GPU, DLA(Deep Learning Accelerator), 영상 신호 처리(ISP) 유닛이 하나의 칩에 통합되어 있어, PC 없이도 카메라 입력부터 딥러닝 추론까지 한 보드에서 끝낼 수 있습니다. 다만 발열·전력 제한으로 최상위 PC/서버 GPU 대비 절대 성능은 낮으므로, 검사 항목이 많고 무거운 모델이 필요하면 워크스테이션/서버와의 성능 차이를 사전에 벤치마크해봐야 합니다.

⑤ 워크스테이션 PC (Workstation PC)

  • 성능과 비용의 균형을 맞출 때 다중 카메라 시스템 구축에 선호되는 선택지
  • 대표 프로세서 : AMD Threadripper PRO, Intel i9

⑥ 엔터프라이즈 서버 (Enterprise Server)

  • 고속 다중 카메라 시스템에서 처리 노드로의 픽셀 데이터 배포 관리에 탁월한 성능 제공
  • 이미지 처리 워크로드에도 높은 성능을 발휘
  • 대표 프로세서 : AMD EPYC, Intel Xeon

⑦ 클라우드 기반 시스템 (Cloud-based System)

  • AI·딥러닝을 활용하는 머신 비전 애플리케이션의 복잡성이 증가할수록, PC 기반 시스템만으로 처리 요구사항을 충족하기 어려울 수 있음
  • 클라우드 기반 플랫폼은 처리를 탄력적으로 확장할 수 있는 리소스를 제공
  • 대표 플랫폼 : AWS, Azure, Google Cloud

3. GPU 가속과 딥러닝 추론

  • 단순 이진화·엣지 검출·블롭 분석 같은 전통적 알고리즘은 CPU만으로도 충분한 경우가 많지만, 딥러닝 기반 검출·분류 모델은 GPU 가속이 사실상 필수
  • NVIDIA GPU 환경에서는 CUDA(병렬 연산) → cuDNN(딥러닝 연산 최적화) → TensorRT(추론 전용 최적화)로 이어지는 가속 스택을 사용
  • TensorRT는 레이어 퓨전(Layer Fusion), 정밀도 보정(FP32→FP16/INT8 양자화), 하드웨어별 커널 자동 튜닝을 통해 동일 모델 대비 2~5배의 추론 속도 향상을 제공
개발자 노트 — 딥러닝 모델을 도입할 때는 학습(Training) 단계보다 실제 라인에 배포되는 추론(Inference) 단계의 지연 시간(Latency)이 더 중요합니다. 동일 모델이라도 FP32 그대로 배포하는 것과 TensorRT로 INT8 양자화해 배포하는 것은 처리 속도 차이가 크므로, 목표 사이클 타임을 먼저 정하고 그에 맞는 최적화 수준을 결정하는 순서로 접근하는 것이 좋습니다.

4. 스토리지 & 데이터 처리량 설계

  • 실시간 판정 결과만 저장하는 시스템과, 원본(Raw) 이미지를 전량 기록해야 하는 시스템은 요구되는 저장 장치 성능이 크게 다름
  • 원본 이미지를 모두 저장해야 한다면, 카메라 페이지의 대역폭 계산식으로 구한 초당 데이터량이 저장 장치의 쓰기 속도(연속 쓰기 기준)를 초과하지 않는지 확인 필요
  • 고속·다중 카메라 시스템에서는 일반 SATA SSD로는 부족할 수 있어, NVMe SSD 또는 RAID 구성을 검토
필요 저장 대역폭(MB/s) = 카메라 대수 × (해상도 × 비트심도 × FPS) ÷ 8
개발자 노트 — "GPU는 충분한데 왜 프레임이 밀리지?"의 원인이 저장 장치 쓰기 병목인 경우가 의외로 많습니다. 특히 여러 대의 카메라로 원본 이미지를 동시에 로깅하는 검사 데이터 축적 시스템에서는 스토리지 대역폭을 GPU/CPU 성능만큼 중요한 설계 요소로 다뤄야 합니다.

플랫폼성능내구성비용대표 용도
소비자 PC중~상 (구성 자유)낮음낮음랩실 검증, 소규모 라인
산업용 PC중~상높음 (방진/내진동)중간공장 현장 상시 가동 라인
비전 컨트롤러중간높음중간~높음PLC 연동, I/O 제어가 많은 설비
임베디드 시스템중간 (전력 제한)중간~높음낮음~중간이동체 탑재, 소형 장비 내장형
워크스테이션 PC상중간중간~높음다중 카메라, 개발/검증 서버

클라우드 기반탄력적 확장해당없음(원격)사용량 기반대용량 AI 학습, 비실시간 분석

6. 머신비전 개발자를 위한 컴퓨팅 플랫폼 선정 체크리스트

  • 처리 알고리즘 종류 — 전통 알고리즘 위주인지, GPU 가속이 필요한 딥러닝 모델인지
  • 목표 사이클 타임 — 카메라 대역폭·프레임레이트 대비 실제 처리·저장까지 끝내야 하는 시간 여유
  • 설치 환경 — 분진·진동·온도 등 산업 환경 내구성이 필요한지, 사무 환경 수준으로 충분한지
  • 확장성 — 향후 카메라 대수·검사 항목이 늘어날 가능성과 그에 따른 여유 성능·슬롯
  • PLC/설비 연동 — 아날로그·디지털 I/O, RS232/485, Modbus 등 설비 제어 신호와의 직접 연동 필요 여부
  • 데이터 관리 — 원본 이미지 전량 저장 여부에 따른 스토리지 대역폭·용량 설계
